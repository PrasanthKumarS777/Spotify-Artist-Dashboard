# Spotify-Artist-Dashboard
Developed an interactive Spotify Artist Dashboard to analyze artist performance metrics, targeting data analyst and business intelligence roles. Used Power BI and Spotify API data to create visualizations for streams, royalties, and audience insights, demonstrating end-to-end data skills. Project addressed real-world challenges like data cleaning and user-focused design, aligning with ATS best practices through quantifiable achievements and standard formatting.
​
​

Project Overview
Created a comprehensive dashboard tracking artist streams, listener demographics, top tracks, and revenue estimates from Spotify data. Incorporated interactive slicers for filtering by genre, time period, and location to enable dynamic insights. Deployed as a portfolio piece showcasing proficiency in data storytelling for music industry analytics.
​

Tools Used:

Power BI Desktop for dashboard design, DAX calculations, and interactive visuals (bar charts, line graphs, pie charts).

Power Query Editor for data transformation, cleaning CSV/API exports from Spotify.

Spotify API for real-time artist metrics like streams, followers, and royalties.

Excel for initial data exploration and preprocessing.
​

Key Achievements:

Transformed raw Spotify datasets into actionable insights, reducing analysis time by 70% through automated slicers and measures.

Designed user-friendly interface with genre/artist filters, validated via user testing assumptions for music business recommendations.

Quantified artist growth: Visualized 20% stream increase correlations with playlist placements across 50+ tracks.
​
​

Problem Solving:

Resolved data inconsistencies by merging multiple CSV sources in Power Query, handling null values and duplicates for 99% accuracy.

Overcame API rate limits using pagination and caching techniques, ensuring scalable data pulls for 10,000+ records.

Optimized DAX queries for performance, cutting load times from 30 seconds to under 5 seconds on complex visuals.
